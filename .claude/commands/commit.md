커밋 규칙을 준수하면서 git commit을 수행합니다.

## 커밋 메시지 형식

```
{타입}: {제목}

- 추가 설명 1
- 추가 설명 2
```

- 타입과 콜론 사이 공백 없음, 콜론 뒤 공백 1개
- 제목과 본문 사이 빈 줄 1개
- 본문은 `-`로 시작하는 bullet point (선택사항)

## 허용 타입
- `feat:` — 새 기능
- `fix:` — 버그 수정
- `refactor:` — 동작 변경 없는 리팩토링
- `chore:` — 빌드, 설정 등 기타

## 절차

1. `git status`와 `git diff --cached --stat`으로 staged 변경 내역 확인
2. 변경 내역을 바탕으로 적절한 타입 선택 후 커밋 메시지 작성
3. 메시지 형식 예시:
   ```
   feat: 로그인 API 추가

   - JWT 토큰 발급 로직 구현
   - 이메일/비밀번호 유효성 검사 추가
   ```
4. `Co-Authored-By` 없이 커밋 실행
5. 커밋 완료 확인
