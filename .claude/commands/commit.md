커밋 규칙을 준수하면서 git commit을 수행합니다.

## 커밋 메시지 형식

```
{타입}: {제목}

- 추가 설명 1
- 추가 설명 2
```

- 타입과 콜론 사이 공백 없음, 콜론 뒤 공백 1개
- 제목과 본문 사이 빈 줄 1개
- 본문은 `-`로 시작하는 bullet point (선택사항)
- 언어: 한국어 권장, 기술 용어·고유명사는 영어 사용

## 허용 타입
- `feat:` — 새 기능
- `fix:` — 버그 수정
- `refactor:` — 동작 변경 없는 리팩토링
- `chore:` — 빌드, 설정 등 기타
- `docs:` — 문서 변경
- `test:` — 테스트 추가/수정
- `style:` — 코드 포맷, 세미콜론 등 로직 변경 없는 스타일

## 절차

1. `git status`와 `git diff --cached --stat`으로 staged 변경 내역 확인
2. 변경 내역을 바탕으로 적절한 타입 선택 후 커밋 메시지 작성
3. `Co-Authored-By` 없이 커밋 실행
4. 커밋 완료 확인
